Help

Unless otherwise specified, _identifiers_ are whatever you want.

Macros

!list:_entity name_!                  boilerplate entity
or
!attr:( iat | uat | str | word )!     adds attribute
or
!_entity_:_entity_!                   creates associate entity

Types

type is:  ( ++             | int     | bool     | str    | ts        | time | date |  bit  | char      | dec     | real | float | money )
or
type is:  ( auto increment | integer | boolean  | string | timestamp | time | date |  bit  | character | decimal | real | float | money )

Options

range  is:  [ digit ]..[ digit ] 

unique is:  unique[ :_group_ ]

option is:  ( _range_  | d        | p       | s      | r        | u        )
or
option is:  ( _range_  | default  | primary | system | required | _unique_ )

Attribute

ref is:  ( _entity_ | _schema_._entity_ )

attribute is:  _name_ as _type_ [ with _option_ [ , ... ] ] 
or
attribute is:  @_ref_ [ as _alias_ ] [ with _option_ [ , ... ] ]

Scheme

# _schema_

## _entity_
- _attribute_

Example

# Kitchen

## Ingredient
- id          as ++
- name        as str  with required, 3..30, unique
- eol         as date with required
- rare        as bool

## Food
- id          as ++
- name        as str  with required, 3..30, unique

## Recipie
- @food               with required, primary
- @ingredient         with required, primary
- amount      as int  with required

# Resturant

## Customer
- id          as ++
- first name  as str  with required, 3..30, unique:fl
- last name   as str  with required, 3..30, unique:fl, unique:ldob
- dob         as date with unique:ldob
- last visit  as ts   with required, default:now

## Order
- @customer           with required, primary
- @recipie            with required, primary
- inserted at as ts   with required, system, default:now
