Guide

The blueprint editor has a markdown inspired syntax, making it easier to rapidly prototype ideas.

Folders

Firstly, create a folder (schema) 

#  [ identifier ]

Blueprints

Secondly, create a blueprint (entity / table) 

##  [ identifier ]

Attributes

Finally, add attribute (properties / columns) 

-  [ identifier ] as [ type ] with [ options ]

Shorthand to referencing other blueprints (identifier must already exist) 

-  @[ identifier ] with [ options ]

Options

Note that all options can be truncated

  • [ required ] must have a value, cannot be null
  • [ unique ]:[ identifier ] must be unique for matching identifier(s) (identifier is optional)
  • [ primary ] this is a primary identifier.
  • [ system ] excludes it from certain things, like needing passed in on creation.

Types

  • [ date ] a calender date, YYYY-MM-DD
  • [ time ] a point in time HH:MM:SS
  • [ timestamp | ts ] includes date and time YYYY-MM-DD HH:MM:SS
  • [ string | str ] a word or phrase
  • [ bit ] 0 or 1
  • [ character | char ] a single character or letter
  • [ decimal | dec ] a decimal point number
  • [ real ] a real number
  • [ float ] a float number
  • [ integer | int ] a signed (+/-) integer
  • [ boolean | bool ] true or false
  • [ money ] a monetary value

Special Types

  • [ auto increment | ++ ] is a unique integer that increments. Automatically gains [ primary & required & unique & system ]
  • [ reference | ref ] indicating another blueprint

Macros

So you scrolled all the way down here? Check this out!

Wrapping a new line in ! runs a macro. The first one starts it, the second one runs it.

Associative Entities

A common need is many-many relationships. For example, a student can be enrolled in many courses, and a course can have many students. Use this to relate 2 pre-existing blueprints.

![ identifier ]:[ identifier ]!

List

Another common need is a list of things to choose from, categorical data. For example, dropdown lists in web design. Use this to make a new blueprint with id and name attributes. 

!list:[ identifier ]!

Attributes

Several attributes make re-appearances throughout different blueprints. These can help you from having to write the same thing over and over. Or just copy-paste I guess... you choose.

  • iat inserted at timestamp
  • uat updated at timestamp
  • word | string | str unique string with upper limit 
!attr:[ prebuilt identifier ]!
