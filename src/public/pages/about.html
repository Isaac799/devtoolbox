Development Stage

Note this application is currently in early development. Outputs are largely untested. For now a lot of what I have is just proof of concept. Plus most of the outputs are based on my limited syntax knowledge and off the top of my head; prone to error and not verbose. This project is constantly being updated and undergoing near daily changes.

Restrictions & Removals

  • Endpoint generation is restricted to single primary keys. For best results use + option with the table. This helps me make a RESTful api.
  • Previously attempted join logic is removed and on pause while I lock down the easier stuff.
  • The NodeJS Express endpoints are temporary removed as I focus my time on the Go api.
  • Removed the "read options" for now in favor of the full read.

PostgreSQL

Lots of good progress, recently added unique indexes on foreign keys. Its been a while since I tested its outputs.
  • The update procedure ignores attributes marked as readonly
  • Two read functions are generated:
    1. Single is basic, letting you read by primary key
    2. Many (plural form) gives all the data

Go Api

A work in progress too, but most promising. As of testing the code runs, but thats as far as my testing has gone. For now I took the sql queries and included them into the Go code to remove the need for stored procedure or functions. Plans to add a toggle may follow, not sure yet.
  • RESTful api design
  • Endpoints grouped by schema to give it more context
  • GET, PUT, POST, DELETE mirroring postgresql table structures
  • GET params are parsed into appropriate type before attempting postgresql call
  • Error handling for pretty much every step
