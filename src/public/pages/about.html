Development Stage

Note this application is currently in early development. Outputs are largely untested. For now a lot of what I have is just proof of concept. Plus most of the outputs are based on my limited syntax knowledge and off the top of my head; prone to error and not verbose.

PostgreSQL

Currently testing the outputs on this one. Lots of good progress, most outputs seem to run and are able to be called but testing it manually is a chore. No promises tho.
  • All aspects (entities, functions, procedures) take into account a composite primary key
  • The update procedure ignores attributes marked as readonly
  • Two read functions are generated:
    1. Single is basic, letting you read by primary key
    2. Options gives a trimmed down list if things in the table
      • automatically done if an attribute is named: name, label, value, code, title, or tag.
      • aimed to help with dropdown list / selection needs

Go Api

A work in progress too, but most promising. While go vet works, I have no clue if it actually works. For now I took the sql queries and included them into the Go code to remove the need for stored procedure or functions. Plans to add a toggle may follow, not sure yet. For now all inputs are passed in via the body, except reads which are attempted to be parsed from query params. May be looking to utilize the path instead for a more "restful" structure.
  • Endpoints grouped by schema to give it more context
  • GET, PUT, POST, DELETE mirroring postgresql table structures
  • GET params are parsed into appropriate type before attempting postgresql call
  • Error handling for pretty much every step

Node JS Express

This is mostly proof of concept, endpoints likely will work, but postgres configuration will need some tweaking.
  • GET, PUT, POST, DELETE endpoints that utilize the postgresql functions and procedures
  • Handholding when making requests, checking for things missed that it expects on a request
  • Error handling for failed queries

Html

This is mostly proof of concept too, very little time spent so far.
  • Form for each CRUD operation needed that aligns with all endpoints created by Node JS Express
