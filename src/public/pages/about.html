Development Stage

Note this application is currently in early development. Outputs are largely untested. For now a lot of what I have is just proof of concept. Plus most of the outputs are based on my limited syntax knowledge and off the top of my head; prone to error and not verbose.

PostgreSQL

Currently testing the outputs on this one. Lots of good progress, most outputs seem to run and are able to be called. No promises tho.
  • All aspects (entities, functions, procedures, joins...) take into account a composite primary key
  • The update procedure ignores attributes marked as readonly
  • Up to three read functions are generated.
    1. Single is basic, letting you read by primary key
    2. Options gives a trimmed down list if things in the table
      • automatically done if an attribute is named: name, label, value, code, or tag.
      • aimed to help with dropdown list / selection needs
    3. Joined recursively grabs foreign attributes too
      • Automatically generated if a foreign key is present
      • Limited for self references
